# Behavior Driven Development(행위 주도 개발, BDD)

- 코드가 어떻게 동작하느냐에 초점을 맞추는것이 아니라 사용자 관점에서 시스템이 어떻게 행동해야하는가에 초점을 맞추는 개발 방법
- 사용자가 어떤 상황에서 어떤 행동을 했을때, 시스템은 어떤 결과를 보여줘야 하는가? 

```text
상황: 사용자가 상품 상세페이지에 있다.
행동: 사용자가 장바구니 담기 버튼을 눌렀다.
결과: 해당 상품이 장바구니에 추가되어야 한다.
```

## BDD가 나온 이유

기획자, 개발자, QA 

기획자: 회원은 로그인하고 주문내역 조회될 수 있어야 해요

개발자:
OrderController -> /orders GET API
JWT 토큰 파싱 -> memberId 추출
OrderRepository -> 주문 목록 조회하면 OK

QA:
로그인한 사용자로 주문 목록페이지 접근시 보인의 주문보이나 확인해봐야지 


## BDD 구성
Given / When / Then

```text
Given: 어떠한 상황이 주어졌을 때

When: 사용자가 어떤 행동을 하면

Then: 어떤 결과가 나올것이다.
```

```gherkin
Feature: 로그인 

  Scenario: 올바른 아이디와 비밀번호로 로그인한다.
    Given: 가입된 회원이 존재한다.
    When: 회원이 올바른 계정과 비밀번호로 로그인을 요청한다.
    Then: 로그인에 성공한다.
    And: 엑세스 토큰이 발급된다.
```
